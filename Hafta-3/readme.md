## 3. Hafta: Uygulama Geliştirme Süreci, Git Entegrasyonu ve Docker Kullanımı
Bu hafta, önceki haftalarda edinilen teorik altyapının pratiğe dökülmesi hedeflenmiş ve yazılım geliştirme sürecinde kullanılan araçların entegre şekilde uygulanmasına odaklanılmıştır. Özellikle konteyner tabanlı geliştirme ortamı ve versiyon kontrol sistemlerinin aktif kullanımı üzerine çalışmalar gerçekleştirilmiştir.
### Yapılan Çalışmalar
Bu doğrultuda Docker altyapısı kullanılarak PHP tabanlı bir başlangıç projesi oluşturulmuştur. Geliştirme ortamı Docker konteynerleri üzerinden yapılandırılmış ve uygulamanın temel giriş noktası olarak index.php dosyası hazırlanmıştır. Böylece PHP uygulamalarının izole ve taşınabilir ortamlarda çalıştırılması deneyimlenmiş, servislerin konteyner yapısı içerisindeki işleyişi gözlemlenmiştir.
Projenin sürüm yönetimi Git aracılığıyla sağlanmıştır. Visual Studio Code terminali üzerinden proje dizininde Git deposu oluşturulmuş; git init, git add, git commit ve git push komutları kullanılarak dosyalar uzak depo ortamına aktarılmıştır. Commit işlemleri sırasında açıklayıcı ve sistematik mesajlar yazılmasına özen gösterilmiş, böylece sürdürülebilir ve düzenli bir versiyon kontrol süreci oluşturulmuştur
Proje sürecinde karşılaşılan teknik hatalar ve yapılandırma problemleri ekip içerisinde değerlendirilmiş, iletişim ve koordinasyon amacıyla Discord platformu aktif olarak kullanılmıştır. Bu süreç, hem teknik problem çözme hem de ekip içi iş birliği açısından katkı sağlamıştır.
Versiyon kontrol yapısında ana dal (main branch) üzerinden ilerlenmiş, dosya organizasyonu ve commit düzeni belirli bir standarda göre sürdürülerek düzenli bir depo mimarisi oluşturulmuştur.
### Öğrenilen Bilgiler
Docker konteynerleri üzerinde PHP uygulaması çalıştırma ve yapılandırma
Git komutlarının terminal üzerinden etkin kullanımı
GitHub üzerinde uzak depo yönetimi
Anlamlı commit mesajlarının proje sürdürülebilirliğindeki rolü
Teknik ekip iletişimi ve hata çözüm süreçlerinin yönetimi
